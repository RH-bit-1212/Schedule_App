# アプリ名：  習慣・スケジュール管理アプリ  

## 1. アプリ概要

本アプリは、**1日分の習慣タスクおよびスケジュールを分単位で管理・記録・可視化するための個人向けWebアプリ**です。  
対象ユーザーは開発者本人のみを想定しており、日々の生活リズムや作業計画を効率的に管理することを目的としています。

### 目的
- 1日のスケジュールを事前に整理し、実行状況をリアルタイムで把握する
- 毎日ほぼ固定の行動（習慣）と、日によって変化する予定（スケジュール）を明確に分離する
- スケジュール作成・修正にかかる手間を最小限に抑える

### 特徴
- **習慣（ルーティン）とスケジュールを分離して管理**
  - 習慣：毎日ほぼ同じ時間・内容で実施する行動
  - スケジュール：日によって内容や時間が変化する予定
- 習慣は一度登録すれば基本的に変更不要
- その日のスケジュールのみを追加・修正することで、1日の計画を素早く作成可能

### 想定される利用例
- 筋トレ、勉強、作業時間など、**毎日決まった時間に行う行動**は「習慣」として登録
- 買い物、病院、外出、突発的な予定など、**日ごとに変わる予定**は「スケジュール」として追加
- 習慣を崩さずに、可変的な予定のみを調整できるため、日々の計画負荷を軽減

### 狙い
- 毎日のスケジュール作成を「ゼロから考える作業」にしない
- 生活リズムを可視化し、継続的な自己管理を可能にする
- 個人用途に特化することで、機能をシンプルかつ実用的に保つ

### 裏の狙い
- このプログラム自体は、シンプルかつ最小限なスケジュール管理アプリ
- これをベースに再利用・改良することが可能(複数選択の岐路をもつようなアプリを第一目標)
- 複数人共有用、日付の追加、会社用、他APIとの連携、AI活用etc

## 2. 機能一覧
- 一覧表示
- 新規追加
- 詳細表示
- 更新
- 削除
- 複数選択・削除
- スケジュールのみ/習慣のみを全削除
- 表示形式切り替え
- 完了状態（done）の切り替え
- 時間帯・完了状態・キーワードなどによるフィルタリング
- 現在進行中タスクの自動抽出し、メイン画面に表示

- Vue.js 3
- Python 3.10+
- FastAPI
- SQLite

## 4. セットアップ
### Frontend
cd api-demo
npm install
npm run serve

ブラウザで以下にアクセスする。 http://localhost:8080

### Backend
cd fastapi-app
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

API が以下で起動する。 http://localhost:8000

- GET /tasks

- POST /tasks
- PUT /tasks/{id}
- DELETE /tasks/{id}

## 6. 保存戦略と理由
選択した保存方式：SQLite

理由：データ件数は1日分のスケジュール量であるため、多くても50件以下になるだろう。また、各データは「タイトル・メモ・開始時刻・終了時刻・タイプ・重要度・完了状態」と、構造自体は変化しにくいものであり、整合性保証などを重視した結果、SQLiteを採用した。逆にLocalStorageを採用しない理由は、今回のアプリをベースに、今後、日付やユーザー項目を追加など、永久保存でなければいけない項目を追加する可能性があり、ブラウザ依存を避けるため。


## 7. 既知の問題・注意点

- 本アプリは個人利用を前提としており、同時アクセスや複数ユーザー利用は想定していない
- 認証・認可機能は未実装であり、外部公開時はアクセス制御が必須
- 入力値のバリデーションは最低限の実装に留まっている
- 不正な時刻指定（開始 > 終了）などの論理チェックは完全ではない
- 長時間稼働・大量データを想定したパフォーマンス検証は未実施
- データのバックアップ・自動復旧機構は未実装
